# Skywalker 框架插件与模块缺口清单及落地步骤

## 1. 当前现状盘点

基于当前代码与配置，插件能力现状如下：

| 插件                  | 当前状态   | 已有模块                                         | 结论                                      |
| --------------------- | ---------- | ------------------------------------------------ | ----------------------------------------- |
| SSFPlugin_Network     | 已投入使用 | SSFModule_NetworkServer, SSFModule_NetworkClient | 基础可用，已具备协议/路由/会话/黑名单能力 |
| SSFPlugin_CommandLine | 已投入使用 | SSFModule_OS                                     | 仅提供命令行/系统基础能力                 |
| SSFPlugin_DB          | 空壳       | 无                                               | 缺少数据访问与持久化模块                  |
| SSFPlugin_Actor       | 空壳       | 无                                               | 缺少实体生命周期管理模块                  |
| SSFPlugin_Level       | 空壳       | 无                                               | 缺少场景/房间管理模块                     |
| SSFPlugin_Profiler    | 空壳       | 无                                               | 缺少性能采集与指标上报模块                |
| SSFPlugin_AI          | 空壳       | 无                                               | 缺少 AI 决策与行为调度模块                |
| SSFPlugin_Video       | 空壳       | 无                                               | 缺少回放录制与回放模块                    |

结论：当前框架的“网络收发层”已基本成型，但“业务承载层、数据层、运维层”模块尚未补齐。

---

## 2. 必要缺口（按优先级）

以下为建议优先补齐的必要模块（先保证服务器能稳定承载业务）：

### P0（必须先做）

1. 认证与会话模块（建议挂在 Network 或新增 Auth 插件）
   - 模块建议：SSFModule_Auth
   - 目标：登录鉴权、Token 校验、重复登录踢线、会话续期。

2. 数据访问模块（DB 插件内）
   - 模块建议：SSFModule_DBAccess
   - 目标：统一 DB 连接池、读写接口、失败重试、超时控制。

3. 玩家数据模块（DB 插件或 Actor 插件）
   - 模块建议：SSFModule_PlayerRepository
   - 目标：玩家基础信息加载/保存、在线脏数据落盘。

4. 世界/房间管理模块（Level 插件内）
   - 模块建议：SSFModule_WorldManager
   - 目标：房间创建销毁、玩家进出、基础广播。

### P1（强烈建议）

5. Actor 生命周期模块（Actor 插件内）
   - 模块建议：SSFModule_ActorRuntime
   - 目标：玩家/NPC 实体创建、Tick、销毁与事件派发。

6. 观测与健康检查模块（Profiler 插件内）
   - 模块建议：SSFModule_Metrics, SSFModule_HealthCheck
   - 目标：QPS、在线数、延迟、错误率、健康状态输出。

7. 管理命令模块（CommandLine 插件内）
   - 模块建议：SSFModule_AdminCommand
   - 目标：在线运维命令（踢人、封禁、热重载配置、统计查看）。

### P2（按业务需要）

8. AI 运行时模块（AI 插件内）
   - 模块建议：SSFModule_AIRuntime
   - 目标：行为树/状态机调度、AI Tick 配额。

9. 回放模块（Video 插件内）
   - 模块建议：SSFModule_ReplayRecorder, SSFModule_ReplayPlayer
   - 目标：对战过程录制、离线回放、问题复盘。

## 3. 分步骤落地计划（可直接执行）

## 第一步：先补 P0 最小闭环（建议 1~2 周）

### 3.1 落地模块

- SSFModule_Auth
- SSFModule_DBAccess
- SSFModule_PlayerRepository
- SSFModule_WorldManager

### 3.2 关键动作

1. 在对应插件 Install/Uninstall 中注册/反注册上述模块。
2. 在 ServerPlugin.skywalkerC 中按环境开启这些模块。
3. 打通最小业务链路：登录 -> 加载玩家数据 -> 进入房间 -> 心跳保活 -> 断线保存。
4. 对每个模块补单元测试（遵循你项目“代码必须有测试”的规范）。

### 3.3 验收标准

- 单客户端可完整登录并进入场景。
- 断线后重连可恢复会话或重登。
- 玩家数据可落库并在重启后恢复。
- 所有新增测试可编译通过。

---

## 第二步：补 P1 运维与稳定性（建议 1 周）

### 4.1 落地模块

- SSFModule_ActorRuntime
- SSFModule_Metrics
- SSFModule_HealthCheck
- SSFModule_AdminCommand

### 4.2 关键动作

1. 增加周期统计：在线连接、在线玩家、消息吞吐、平均延迟。
2. 增加健康检查输出（可先落日志，再扩展到 HTTP/命令行查询）。
3. 实现基础管理命令：
   - reload_config
   - kick_player
   - ban_ip / unban_ip
   - show_stats

### 4.3 验收标准

- 服务异常时可在 1 分钟内通过日志或命令定位。
- 运维命令可动态生效，无需重启。

---

## 第三步：补 P2 业务增强能力（按项目节奏）

### 5.1 落地模块

- SSFModule_AIRuntime
- SSFModule_ReplayRecorder
- SSFModule_ReplayPlayer

### 5.2 关键动作

1. 先实现 AI Tick 配额与基础行为树接口。
2. 先实现关键消息录制（输入、状态同步、关键事件）。
3. 提供最小回放工具链（按会话ID读取回放并离线播放）。

### 5.3 验收标准

- AI 不会挤占主线程 Tick。
- 指定对局可回放并用于问题复盘。

## 5. 建议执行顺序（最短路径）

1. 先做 Auth + DBAccess + PlayerRepository（解决登录和数据闭环）。
2. 再做 WorldManager（让“在线玩法”有承载容器）。
3. 然后补 Metrics + AdminCommand（提升可运维性）。
4. 最后按业务价值补 ActorRuntime / AI / Replay。
